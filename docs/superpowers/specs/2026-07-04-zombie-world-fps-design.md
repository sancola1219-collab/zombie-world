# 《殭屍世界：黑鴉市事件》設計文件

日期：2026-07-04
狀態：已由使用者核准設計方向（三章戰役、平衡型、中度血腥、桌面優先＋手機可玩、技術方案 B）

## 1. 概述

第一人稱 3D 射擊冒險解謎遊戲，致敬經典生存恐怖系列（洋館、警局、地下實驗室、打字機存檔、草藥、心電圖狀態、不死追跡者），劇情與名稱全部原創，無任何註冊商標文字或角色名，可安全公開發佈。

- 平台：純前端靜態網頁，發佈至 GitHub Pages（帳號 sancola1219-collab，路徑 `…/zombie-world/`）
- 技術：Three.js（CDN import map、鎖定版本）、ES modules 多檔結構、全程序化資產（Canvas 貼圖、WebAudio 合成音效），除 Three.js CDN 外零外部資源請求
- 語言：全繁體中文（UI、文件、劇情）
- 目標長度：三章共約 45–60 分鐘通關

### 非目標（明確不做）

- 不使用外部 3D 模型 / 貼圖 / 音檔素材
- 不做多人、成就系統、多語系
- 不做即時陰影以外的高階渲染（維持低多邊形復古恐怖風格）
- 不做完整 navmesh 尋路（房間連通圖＋直線導引即可）

## 2. 劇情與世界觀

2003 年，山城「黑鴉市」。表面是製藥龍頭的「晨星製藥」在山區祕密設施研發生物兵器「N 病毒」，意外外洩。主角**凱倫**（無語音、第一人稱不露臉）是黑鴉市警局特勤小組（S.O.G.）隊員，奉命進山調查登山客連環失蹤案，隊伍遭襲失聯，獨自逃入山中洋館——遊戲從此開始。

劇情推進以「收集文件」為主（守衛日記、研究員筆記、內部備忘錄），過場用全螢幕文字卡＋音效呈現（不做 3D 過場動畫）。

## 3. 章節設計

### 第一章〈月影洋館〉（致敬初代）

**結構**：1F（玄關、大廳、東走廊、餐廳、廚房、鋼琴房、西走廊、書房、標本室）、2F（時鐘間、客房、主臥、迴廊）、B1（地下通道、纜車站）。

**推進鏈**（依賴順序）：

1. 玄關→大廳。東翼門「月之鎖」、西翼門「星之鎖」上鎖；東走廊可通行
2. 東走廊：**第一隻殭屍緩緩回頭**開場事件（系列標誌鏡頭，強制近距離遭遇後給手槍）
3. 廚房：取得**打火機**、綠草藥、樂譜散頁（上）
4. 餐廳：用打火機點燃壁爐→暗格取得**月之鑰**
5. 月之鑰開 2F 時鐘間與客房：
   - 時鐘間：依守衛日記線索「獵犬吠叫的時刻」把時鐘調到 3:00 →**蛇徽章**
   - 客房：手槍彈、樂譜散頁（下）、守衛日記①（致敬「癢…吃了」的原創版本）
6. 月之鑰開鋼琴房：集齊樂譜彈奏→暗門開啟→**鷹徽章**＋**星之鑰**
7. 星之鑰開西翼：
   - 西走廊：**殭屍犬破窗**驚嚇橋段（2 隻）
   - 書房：保險箱密碼線索文件＋晨星偽裝設施揭露文件
   - 標本室：依文件將三座獸類標本轉向正確方位→**狼徽章**
   - 2F 主臥：保險箱（書房線索密碼）→**鹿徽章**＋**霰彈槍**
8. 四徽章嵌入大廳地板浮雕→地下階梯開啟
9. B1 地下通道：精英怪**獵痕者**首次遭遇（可戰可逃）→纜車站搭纜車下山，章末文字卡

**敵人配置**：殭屍約 12–15、殭屍犬 3、獵痕者 1。打字機存檔點：大廳、2F 迴廊。

### 第二章〈黑鴉市警察局〉（致敬二代，序段致敬三代）

**序段**：封鎖街道——線性市區路段，火光、車輛殘骸、大量殭屍，教學「跑過去比打光子彈聰明」，抵達警局正門。

**警局結構**：1F（大廳、檔案室、拘留所、東翼暗廊）、2F（圖書館、局長室、武器庫、觀景廳）、B1（地下停車場）。

**推進鏈**：

1. 大廳**女神像**：需嵌入紅・藍・綠三寶石
2. 檔案室（開放）：**黑桃鑰匙**＋「局長的紀念日」密碼線索文件
3. 拘留所（黑桃鑰匙）：**綠寶石**（水槽中）＋**紅心鑰匙**＋倖存警員遺言文件
4. 東翼暗廊（停電區，手電筒自動開啟）：**潛伏者**首次遭遇——天花板爬行、聽聲辨位，走路不開槍可避戰
5. 圖書館：書架推移謎（推動書架對齊夾層走道）→**紅寶石**
6. 局長室（紅心鑰匙）：保險箱（紀念日密碼）→**藍寶石**＋**武器庫鑰匙**
7. **取得第二枚寶石時**：不死追跡巨人**「執行者」破牆登場**——不可擊殺（可暫時擊倒 10 秒）、跨房間追擊、緩慢逼近的腳步聲、不進入打字機安全房
8. 武器庫（武器庫鑰匙）：**麥格農**＋彈藥補給
9. 女神像集齊三寶石→吐出**停車場鑰匙卡**
10. B1 停車場：潛伏者伏擊區（2 隻）→鑰匙卡開裝甲門→貨梯直下實驗室，章末文字卡

**敵人配置**：街道殭屍約 15（不必清）、警局殭屍約 10、潛伏者 3、執行者（常駐追跡）。打字機：大廳、2F 觀景廳（安全房）。

### 第三章〈晨星地下實驗室〉（致敬初代／二代結局）

**結構**：接待大廳、配電室、研究區、樣本庫、主控室、培養槽區、貨運月台。

**推進鏈**：

1. 到達時全區停電，僅緊急照明。配電室：**接線謎題**（三條電路各自把線段旋轉接通）→恢復電力
2. 研究區：研究員筆記（化學配方線索）＋ N 病毒真相文件＋執行者來歷文件
3. 樣本庫：依筆記混合試劑 A＋B 製成**強酸觸媒**→溶解主控室生化鎖
4. 主控室：取得**高層權限卡**（開啟培養槽區閘門）＋讀到「緊急自毀協定」文件（伏筆：設施重損時自動啟動）
5. 培養槽區（權限卡）：**執行者最後一次擋路→被破槽而出的原體撕碎**（過場事件，免雙頭戰鬥）
6. **Boss 戰「原體」第一階段**（人形態，弱點背後紅色增生體）
7. 逃至貨運月台：**第二階段**（巨大態堵住月台，遠程觸手＋酸液）→劇情給予**火箭筒**→終結
8. Boss 倒下重創設施→**自毀協定自動啟動，倒數 5 分鐘**：警報紅光、路線塌陷改道，衝回貨梯→直升機救援結局
9. 通關評價（S–D）：依總時間、存檔次數、文件收集率

**敵人配置**：殭屍（研究員）約 8、獵痕者 2、原體（Boss 兩階段）。打字機：接待大廳、主控室。

## 4. 玩法系統

### 操作

| 動作 | 桌面 | 手機 |
|---|---|---|
| 移動 | WASD | 左虛擬搖桿 |
| 視角 | 滑鼠（Pointer Lock） | 右半螢幕拖曳 |
| 射擊 / 瞄準 | 左鍵 / 右鍵 | 射擊鈕（自動輕瞄準輔助） |
| 互動 / 裝填 | E / R | 情境互動鈕 / 裝填鈕 |
| 奔跑 | Shift | 搖桿推到底 |
| 物品欄 / 地圖 | Tab / M | HUD 圖示鈕 |

Pointer Lock 失敗時退化為「按住拖曳」視角模式。手機自動偵測（觸控事件）切換 UI。

### 武器（傷害 / 容量）

小刀（15、無限、近戰）→ 手槍（25、15 發彈匣）→ 霰彈槍（8 顆彈丸 ×12、7 發）→ 麥格農（150 貫穿、6 發）→ 火箭筒（Boss 終結劇情武器）。爆頭 2 倍傷害；殭屍爆頭有機率一擊斃命。彈藥全程稀缺，需撿拾與取捨。

### 生命與治療

HP 0–100。HUD 心電圖：Fine（>66，綠）/ Caution（33–66，黃）/ Danger（<33，紅＋移動 -20%）。綠草藥 +40、紅草藥單用無效、**綠＋紅組合全滿**。急救噴霧（稀有）全滿。

### 敵人 AI（狀態機）

| 敵種 | 行為 | HP |
|---|---|---|
| 殭屍 | 慢速直追、抓咬 20 傷、倒地可再起（爆頭防再起） | 70 |
| 殭屍犬 | 快速、跳撲 15 傷、跳撲後硬直 | 40 |
| 潛伏者 | 天花板潛行、玩家奔跑或開槍時觸發撲擊 25 傷 | 90 |
| 獵痕者 | 精英、高速蛇行接近、跳斬 40 傷 | 200 |
| 執行者 | 不死、跨房追擊（永遠知道玩家所在房間、會開門）、重擊 35 傷＋擊退；集中火力可擊倒 10 秒；不進安全房 | 不可殺 |
| 原體 | 兩階段 Boss，見章節設計 | P1 600 / P2 火箭筒終結 |

尋路：房間連通圖 BFS＋房內直線導引＋牆面滑動，不做 navmesh。

### 謎題原則

所有密碼／方位／時刻類謎題的正確解都定義在關卡資料中，且對應線索文件必須寫明可推得唯一解（不靠暴力嘗試）。所有上鎖門在地圖上標示所需鑰匙圖示。

### 物品欄與道具

8 格限制；關鍵劇情道具（鑰匙、徽章、寶石）不佔格（獨立「重要物品」頁）。支援檢視、使用、丟棄（劇情道具不可丟）、組合（草藥混合、同種彈藥堆疊）。

### 存檔

場景中的**打字機**互動存檔（手動 1 槽）＋每章開頭自動存檔（1 槽）。localStorage key `zombie-world-save`；序列化內容：章節、所在房間、玩家位置與狀態、物品欄、全域旗標集、已殺敵人 ID、已拾道具 ID、統計（遊戲時間、存檔次數、文件收集數）。localStorage 不可用時退化為記憶體存檔並提示。

### 地圖與文件

M 鍵樓層平面圖：已探索房間上色、含未拾道具的房間標記（致敬經典紅/藍房間）。文件收集進閱讀庫，可隨時重讀，收集率計入評價。

js/game/puzzles.js    — 謎題狀態機框架（條件檢查、旗標讀寫、互動介面掛載）

### 核心設計決策

- **狀態與動畫分離**（既有教訓）：所有遊戲邏輯跑固定時步，rAF 只負責渲染；分頁隱藏、rAF 停擺時邏輯不受影響，倒數計時等用邏輯時鐘而非牆鐘
- **ResizeObserver** 監聽容器尺寸（載入時尺寸可能為 0 的既有課題）
- 關卡＝資料：房間為軸對齊盒體＋牆面開口清單，幾何由資料生成；門連接房間並帶鎖 ID；渲染僅當前房間＋相鄰房間（房間級剔除）
- 碰撞：玩家/敵人膠囊體 vs 房間牆線段（2D 平面解算＋高度判定），不做物理引擎
- 事件觸發器：進房 / 拾取 / 旗標變更三類，驅動劇情橋段（殭屍回頭、狗破窗、執行者登場）

### 錯誤處理

- WebGL 建立失敗→顯示不支援提示頁
- Pointer Lock 被拒→自動切換拖曳視角模式並提示
- localStorage 例外→記憶體存檔＋警示「關閉頁面將遺失進度」
- 手機/低效能（幀率偵測）→關閉動態陰影、降低解析度縮放

## 6. 測試策略

- **同步模擬單元驗證**（既有教訓：不依賴瀏覽器計時器）：傷害計算、彈藥/物品欄邏輯、草藥組合、謎題狀態機、存檔序列化往返、章節推進鏈可通性（腳本模擬拾取→開鎖→旗標順序走完全章）
- 每章手動通關檢查表：所有門鎖對應鑰匙、所有謎題可解、觸發器只發動一次、存檔讀檔後狀態一致
- 桌面（滑鼠鍵盤）＋手機（觸控模擬）雙操作驗證
- 發佈後於 GitHub Pages 實際網址驗證（CDN import map、路徑大小寫）
